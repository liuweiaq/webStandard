总体要求

  • 在HTML页面里只写入文档的结构，确保页面结构、样式、行为的分离；
  • HTML文件统一采用UTF-8无BOM编码；
  • 代码缩进的基本单位为4个空格(或一个tab键)，嵌套元素应当缩进一次；
  • 如无特别需求，所有单词一律小写；
  • HTML页面的第一行使用 HTML5 的doctype来启用标准模式，建议使用大写的DOCTYPE；
    • <!DOCTYPE html>
  • 建议在HTML根元素中指定lang属性。

    • 命名规范

  • HTML文件及模板文件名称应符合Camel命名规范。如：index.html；
  • 相关属性值名称(元素id、元素name等)和自定义属性名要求全字母小写，单词之间用下划线(_)分隔。如：btn_cancle；
  • 自定义属性名加前缀data。如：data_order；
  • 命名单词须含义准确，通俗易懂，拒绝使用汉语拼音进行命名；
  • 元素id必须保证页面唯一。

    • head相关说明

  • 明确定义网页标题；
    • <title>网页主题</title>
  • 明确声明字符编码，方便浏览器快速判断页面内容的渲染方式；
    • <meta charset="UTF-8">
  • 如果需要兼容IE，请设置IE兼容模式；
    • <meta http-equiv="X-UA-Compatible" content="IE=Edge">
  • 创建移动设备友好的页面，建议指定页面的viewport；
    • <meta name="viewport" content="width=device-width, initial-scale=1">
  • CSS与采用异步加载时的JavaScript文件均在head中引入，引入时可以不指定type属性；

    • 所有的CSS资源均须在head中引入，因为在页面渲染过程中，新的CSS可能导致元素样式的重新计算和绘制而页面闪烁。

    JavaScript 应当放在页面末尾，或采用异步加载放在head标签中引入，因为将script放在页面中间将阻断页面的渲染。

    text/css 和 text/javascript 是 type 属性的默认值。

    <script src="../require.js" data-main="index"></script>
    <link href="css/bootstrap.min.css" rel="stylesheet">
  • HTML标准模板(示例)
    • <!DOCTYPE html>
    <html lang="en">
        <head>
            <meta charset="UTF-8">
            <meta http-equiv="X-UA-Compatible" content="IE=Edge">
            <meta name="viewport" content="width=device-width, initial-scale=1">
            <title>网页标题</title>
            <link href="css/bootstrap.min.css" rel="stylesheet">
            <script src="../require.js" data-main="index"></script>
        </head>
        <body>
            ...
        </body>
    </html>

    标签规范

  • 自闭合标签尾部不能添加斜线。

    • HTML5中常用的自闭合标签：img, input, link, meta, source, br, hr, param, area, base, col, command, embed, keygen, track。

    Bad Code Good Code
    <img class="oso-avatar" src="..."/> <img class="oso-avatar" src="...">
  • 非自闭合元素不能省略闭合标签。
    • Bad Code Good Code
    <ul>
         <li> first
         <li> second
    </ul>     		<ul>
         <li> first </li>
         <li> second </li>
    </ul>
  • 尽量使用最少的标签，保持最小的复杂度。

    • 在HTML代码中，可以通过迭代和重构来避免多余的父元素。

    Bad Code Good Code
    <span class="oso-avatar">
         <img src="...">
    </span>     		<img class="oso-avatar" src="...">
  • 不能使用table标签来进行布局。
  • HTML标签的使用应该遵循标签的语义。

    • HTML标签语义见附录：HTML5标签列表。

    Bad Code Good Code
    <div onclick="goToRecommendations();">
         All recommendations
    </div>     		<a href="recommendations/">
         All recommendations
    </a>

    属性规范

  • 属性的定义，确保全部使用双引号，不要使用单引号。
  • 可以给元素设置自定义属性来存放与JavaScript交互的数据，其命名应符合相应规范。
  • 为重要图片添加有意义的alt属性，说明图片内容。
    • Bad Code Good Code
    <img src="logo.gif"> <img src="logo.gif" alt="oso">
  • 建议按照固定顺序排列属性，确保代码的易读性。

    • 属性排列的优先级：class—>id, name—>data_*—>src, for, type, href—>title, alt—>role。

    Bad Code Good Code
    <input data_order="12" type="button" id="btn_bill_submit" class="oso-bill-submit" name="bill_submit"> <input class="oso-bill-submit" id="btn_bill_submit" name="bill_submit" data_order="12" type="button">
